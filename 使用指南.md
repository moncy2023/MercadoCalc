# MercadoCalc - 美客多店铺利润核算系统使用指南

**Project Name**: MercadoCalc (Mercado Libre Pricing & Profit Calculator)
**项目简介**：专为美客多（Mercado Libre）卖家设计的全动态、多国实时运费对接的定价与利润核算系统。支持自动反推售价和计算实际利润，帮助卖家快速做出保本与盈利决策。

## 一、环境准备

### 1.1 安装 Python 依赖
```bash
pip install fastapi uvicorn pydantic pymupdf
```

### 1.2 初始化数据库
首次使用前，运行以下命令生成运费数据库：
```bash
python pdf_extractor.py
```
> 该命令会在当前目录生成 `shipping_fees.db`，并自动录入巴西运费数据（25条阶梯记录）。

---

## 二、启动系统

### 2.1 Web 模式（推荐）
```bash
python main.py web
```
启动后在浏览器打开：**http://127.0.0.1:9001**

### 2.2 命令行模式
```bash
python main.py cli
```
从 `config.json` 读取参数，直接在终端输出核算结果。

### 2.3 自定义端口
```bash
python main.py web --port 9000
```

## 三、Web 界面操作

### 3.1 利润核算（首页）

#### 步骤一：填写基础信息
| 字段 | 说明 | 示例 |
|------|------|------|
| 采购成本 (¥) | 产品采购价 + 国内物流费 | 50.0 |
| 抽佣比例 | 美客多平台抽佣，0.15 = 15% | 0.15 |
| 退货率 | 退货损失比例，0.02 = 2% | 0.02 |

#### 步骤二：设置运费参数
| 字段 | 说明 | 示例 |
|------|------|------|
| 国家/地区 | 下拉选择目标国家（默认巴西） | Brazil |
| 汇率换算 | 1 USD = ? RMB，影响运费和售价换算 | 7.2 |
| 是否高于免邮阈值 | 勾选后使用"高于阈值"运费标准，取消勾选使用"低于阈值"标准 | ✅ |
| 商品重量 (g) | 产品重量，单位为**克** | 200 |
| 平台附加费 (¥) | 重量/体积不对时平台收取的额外费用 | 0 |
| 平台运费折减 (¥) | 平台给予的运费优惠减免 | 0 |

> **【总运费】** 会根据以上参数**自动实时计算**，无需手动填写。
> 下方会显示计算明细，例如：`USD 5.1 × 7.2 = ¥36.72`

#### 步骤三：选择计算模式并核算

**模式一：反推售价**（已知目标利润）
1. 选择 "反推售价 (已知目标利润)"
2. 填写目标利润百分比（如 0.20 = 20%）
3. 点击【开始核算】
4. 结果会显示：平台售价（USD）、利润（¥）、利润率等

**模式二：计算利润**（已知售价）
1. 选择 "计算利润 (已知平台售价)"
2. 填写平台售价
3. 点击【开始核算】
4. 结果会显示：实际利润（¥）、利润率等

#### 核算结果说明
| 输出项 | 单位 | 说明 |
|--------|------|------|
| 平台售价 | USD | 建议在美客多上架的价格，括号内附 ¥ 参考值 |
| 【总运费】 | ¥ | 头程+尾程-折减，附折减金额 |
| 固定成本 | ¥ | 采购+物流+附加费的总和 |
| 平台抽佣 | ¥ | 售价 × 抽佣比例 |
| 退货成本 | ¥ | 售价 × 退货率 |
| 利润 | ¥ | 最终利润金额 |
| 利润率 | % | 利润占售价的百分比 |

## 四、运费数据管理

### 4.1 导入新国家的运费标准
如果从美客多官方获取了某国的运费 PDF（纯文本型），可以一键导入：
```bash
python import_pdf.py --country Mexico --pdf mexico_shipping.pdf
```

> ⚠️ **注意**：如果 PDF 是纯图片扫描件（无法选中文字），则无法自动提取。
> 这种情况下需要手动在 `pdf_extractor.py` 中录入数据，参考巴西数据的格式。

### 4.2 手动录入运费数据
通过 Python 命令直接插入：
```bash
python -c "
import database
# insert_rate(国家, 重量下限kg, 重量上限kg, 低于阈值费用USD, 高于阈值费用USD)
database.insert_rate('Mexico', 0.0, 0.5, 3.0, 8.0)
database.insert_rate('Mexico', 0.5, 1.0, 5.0, 12.0)
print('录入成功')
"
```

### 4.3 设置国家免邮阈值
```bash
python -c "
import database
# set_country_metadata(国家, 本币代码, 本币阈值, USD等值)
database.set_country_metadata('Brazil', 'BRL', 79.0, 15.0)
database.set_country_metadata('Mexico', 'MXN', 299.0, 17.0)
print('设置成功')
"
```

### 4.4 查看数据库中已有数据
```bash
# 查看所有国家
python -c "import database; print(database.get_all_countries())"

# 查看某国运费阶梯
python -c "import database; [print(r) for r in database.get_country_rates('Brazil')]"

# 查看某国阈值
python -c "import database; print(database.get_country_metadata('Brazil'))"
```

## 五、配置文件说明（config.json）

配置文件用于 CLI 模式下的默认参数，也是各字段的初始值参考：

```json
{
    "procurement_cost": 50.0,           // 采购成本（¥）
    "manual_final_shipping_fee": 0.0,   // 手动运费覆盖（0=自动计算）
    "exchange_rate_usd_to_rmb": 7.2,    // USD转RMB汇率
    "is_above_threshold": true,         // 是否高于免邮阈值
    "platform_shipping_discount": 0.0,  // 平台运费折减（¥）
    "return_rate": 0.02,                // 退货率 2%
    "commission_rate": 0.15,            // 抽佣比例 15%
    "additional_fee_for_wrong_weight": 0.0,  // 附加费（¥）
    "calculation_mode": 1,              // 1=反推售价, 2=计算利润
    "target_profit_percentage": 0.20,   // 目标利润率 20%
    "platform_selling_price": 100.0,    // 平台售价（模式2用）
    "country_for_shipping": "Brazil",   // 默认国家
    "weight_g": 1000.0                  // 商品重量（克）
}
```

## 六、常见问题

**Q：切换国家后总运费没变？**
A：请确认该国家在数据库中有运费数据。可通过 `/shipping` 页面查询确认。

**Q：PDF 导入失败提示"未匹配到格式"？**
A：该 PDF 可能是纯图片扫描件，无法自动提取文字。需要手动录入数据（见 4.2）。

**Q：如何更新汇率？**
A：直接在 Web 页面的"汇率换算"输入框中修改即可，所有运费和售价会实时重新计算。

**Q：如何添加新国家？**
A：
1. 用 `import_pdf.py` 或手动命令录入该国运费数据（见 4.1/4.2）
2. 用 `set_country_metadata()` 设置该国免邮阈值（见 4.3）
3. 刷新 Web 页面，新国家会自动出现在下拉列表中

## 七、NAS (群晖 DSM 7.2+) 详细部署指南

本项目原生支持 Docker 部署，特别适合部署在群晖等 NAS 设备上，实现全天候随时随地访问。以下是针对 **DS923+ (DSM 7.2.2)** 及以上版本的最详细的“保姆级”部署步骤：

### 1. 前期准备工作
1. **安装 Container Manager**：登录群晖 DSM 网页后台，打开“套件中心”，搜索 `Container Manager`（旧版本叫 Docker）并安装。
2. **确认文件位置**：确保你目前的这个项目文件夹 `1、mercadocalc` 已经存放在 NAS 的某个共享文件夹中（例如你的 `缓冲区` 文件夹）。请记住这个文件夹的具体路径。

### 2. 创建并构建项目 (Container Manager)
1. 打开群晖的 **Container Manager** 套件。
2. 在左侧菜单点击 **项目 (Project)**，然后点击顶部的 **新增 (Create)** 按钮。
3. 在弹出的“创建项目”窗口中，严格按照以下步骤填写基本信息：
   * **项目名称**：填入 `mercadocalc`（注意必须全是小写字母，不能有中文或特殊符号）。
   * **路径**：点击右侧的“设置路径”按钮。在弹出的文件管理器中，找到并选中你存放代码的 `1、mercadocalc` 文件夹（也就是包含 `docker-compose.yml` 文件的那个文件夹），点击底部的“选择”。
   * **来源**：选择 **从现有文件构建 (Build from existing docker-compose.yml)**。
4. **设置网页服务门户**：
   * 点击 **下一步**，此时会弹出“设置网页服务门户”的界面（就是你之前截图的那个界面）。
   * **直接跳过**：什么都不需要填！直接点击右下角的 **下一步**，或者点击窗口右上角的 **X** 关闭弹窗（因为我们在代码里已经自动分配好了 9001 端口，不需要网页服务门户多此一举）。
5. **确认并启动**：
   * 再次点击 **下一步**。
   * 勾选“在向导完成后构建并启动项目”（默认已勾选），点击 **完成**。
6. **等待构建完成**：
   * 系统会自动弹出一个黑色的终端日志窗口，显示正在拉取 Python 镜像并安装运行环境（你会看到类似 `Pulling python:3.10-slim` 的字样）。
   * 这个过程可能会持续 3-5 分钟，取决于你的 NAS 网络速度，请耐心等待。
   * 当看到最后一行提示 `Exit Code: 0`（退出代码：0），就表示构建和启动彻底成功！点击“关闭”按钮。

### 3. 访问并使用你的计算器
1. **内网访问**：打开你的电脑浏览器，输入 `http://你的群晖NAS_IP:9001`（例如 `http://192.168.2.254:9001` 或 `http://192.168.8.249:9001`）。你应该能直接看到美客多运费利润计算器的网页界面了。
2. **外网访问（可选）**：如果你需要出门在外时也能用手机/电脑访问，可以通过 Tailscale 虚拟局域网访问，或者在路由器中将局域网 IP 的 9001 端口映射到公网。

### 4. 日常数据维护（极度重要）
本项目已在 `docker-compose.yml` 中配置了**数据持久化挂载**。这意味着你的核心数据（数据库和配置）和系统环境是分离的。
* **如果要更新汇率或免邮门槛**：直接用电脑在 VS Code 里打开 `1、mercadocalc/config.json` 文件修改，或者直接在 Web 网页界面上修改（网页修改会自动保存到 config.json）。
* **如果要更新官方 PDF 运费表**：直接在电脑上运行 `import_pdf.py`，它会把最新的数据写入 `1、mercadocalc/shipping_fees.db` 数据库。
* **如何让 NAS 上的系统生效新数据？** 你的数据文件是直接实时挂载到 NAS 里的。如果你在电脑上跑了脚本更新了数据库，网页上数据没变，只需要打开群晖 Container Manager，在“**容器 (Container)**”列表中找到 `mercadocalc_app`，右键点击**重新启动**，1秒钟后最新数据就会生效，且永远不会丢失！
